Data Table Directive Guide

##Overview## The rx-data-table directive is designed to pretty much be a catch-all solution for displaying data in a table format in a basic or extremely complex style. It has some basic CSS styling included, but provides hooks for you to attach conditional styles to each row or column.

###Some Key Features###

  • Single or Multi-Column Sorting
  • Column Display Presets
  • Individual Data-Driven Cell Styling
  • CSS FlexBox display for fluid space display

##Column Object Attributes##

Attribute Type Details
checkbox boolean=false If this is true, then the field becomes a checkbox and relies on the dataField property to apply ID Values to the checkboxes.
class string This is a css class that gets applied to the div that wraps the data.
cols number This is the value that will be passed to the flexbox css code. This is a proportional value, so if you put a value of 2 on one column and a value of 1 on another column, the first column will always be twice the size of the second.
dataField string
array.<string>		 This is the data field on the object that this column will display. If it is an array of string, then a line break will be inserted between each evaluated field. If a filter is specified, it will apply itself to ALL of the data fields.
editable string This attribute should be an object describing how you want to handle field editing. The description of the available attributes and configuration can be found here.
emptyValue string If set, the string value of this attribute will be displayed if this field is editable and empty.
filter string If you specify this attribute, then the value will be resolved to the appropriate filter. The filter should expect row, column objects to be passed to it.
filterParameters array This is an optional list of parameters to pass to the filter.
help object If this is specified, then the column will have a help bubble beside the header that can have information populating it. The object MUST have a body attribute which should be the body of the text. It can also have a title attribute that will give the popover a title header. REQUIRES UI Bootstrap Popover Module
hideable boolean=false This is used to determine whether this column can be hidden in the column reordering/hide/display configuration dialog. This doesn't mean that the presets themselves can't hide them, it just prevents the users from hiding it.
icon array.<object> This is a list of objects that will resolve to icons to be displayed. The attributes for the objects are as follows:
  • name: If this is specified, the column uses Font Awesome icons. The name is the name of the Font Awesome icon class.
  • class: If class is specified, then a div is created and the class is applied to it.
  • field: Which field should the icon get its data from.
  • fieldValue: What should the field be compared against to determinewhether or not this particular icon should be shown. If this is not applied, then a truthy check is done on the field.
  • alt: This is the alt text that gets applied to the icon.
  • tooltip: This will use add the ui-bootstrap tooltip module and allow you to have a tooltip for this icon. This should be an object with text as the tooltip text (REQUIRED) and can take placement to place the tooltip according to the module's accepted values. REQUIRES UI Bootstrap Tooltip Module
linkFunction function This is a function that will generate the link for this column.
linkField string This is the data field where a link will be pulled from for the a href link. This is ignored if there is a linkFunction present.
linkTarget string This is the a target that you would like the link to go to.
menu object This attribute should be an object describing how you would like the menu to be displayed. The base object only has two root elements: icon (optional) which is a class for the icon in the menu (defaults to fa-cog), and then items which is the list of Menu Item Objects, the configuration of which can be found here.
ng-class function(row) This is a function that gets passed the row. The return value is expected to be a string that will get applied as a class.
sortField string This is the data field that the column will sort by. If it is not supplied, the column will default to sort by the dataField. If equal to false, sorting will be disabled for this column.
sortInverse boolean=false Should the display of this column's sort be inverted.
title string The column header title. This is required unless the column type is a checkbox.
tooltip string This should be a string value to the data field that you want to populate the tooltip with.

##Inline Table-Cell Editing##

The Data Table Directive has the capability to have individual fields have the ability to be edited. You can even have conditions be applied that make a field editable on rows that only meet certain conditions.

Attribute Type Details
clause function(row) This is a function that should evaluate to a boolean value. The function will get passed the data row object so that you can run your evaluation on it.
method function(column, row, data) This is a function that will do the submission for the field update. The function will get passed the column object, the row data, and the new data value for the row's column. It will only get called if endpoint hasn't been specified.
endpoint object This attribute is used to update the data on the server on save. It is expecting an object with two attributes:
  • url - string This should be a url (absolute or relative) that you will submit the data to. If you put in a field name inside curly-braces, it will resolve them to the row's field values (i.e. '/api/{system}')
  • method string This should be the HTTP method to use. (i.e. GET, PUT, POST, etc.)
data function(row) This is a function that should return a promise or a list of options to display in the typeahead (see the options attribute for structure). It will be passed the data row object.
options array.<object> This attribute will tell the directive to make this editable field a select box and provide options to the user for selection. They are standard select options with value and text attributes:
  • value - string This is the value that will get passed back to the api call.
  • text - string This is the value that the user will see in the select dropdown
nullable boolean This is a boolean value that if set to true will add an X next to the field that will allow users to click and quickly set the values on the item's field empty.
preUpdate function(column, row, newValue) This method will run just before a server update is called. This method must return a value of true or the update will halt.
postUpdateSuccess function(column, row, newValue) This method will run after the update is successfully completed with the server, if it does so.
postUpdateError function(column, row, newValue) This method will run after the update fails to complete with the server, if it does so. It will not rollback any of the changes applied during the updateMethod, unless you specifically add them to this method.

##Contextual Menus##

The Data Table Directive has the capability to have one or more of the columns have a contextual menu with configurable actions and options.

The Menu will take a list of items that you want to show up.

Attribute Type Details
text string This will be the text that will appear for the menu option.
icon string This will be the CSS Class that will be used to apply the icon. It should be a FontAwesome Icon.
class string This will be the CSS Class that will be applied to this menu option. You can use this for styling purposes.
action <a href class="label type-hint
