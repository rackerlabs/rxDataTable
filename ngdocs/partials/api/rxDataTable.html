rxDataTable

Description

Directive that creates a data table with responsive design properties. This requires the following directives:

  • rxPaginate Pagination Directive
  • rxItemsPerPage Items per Page Directive

Usage

as element:
<rx-data-table
       [pager-object="{Object}"]
       list-of-data="{Array.<Object>}"
       [predicate="{Array.string=|string=|boolean}"]
       [row-key="{string}"]
       [row-details="{function}"]
       [row-details-clause="{expression}"]
       item-name="{string='Items'}"
       [notify-duration="{number}"]
       [data-loading="{boolean}"]
       [row-style="{string}"]
       column-presets="{array.<Object>}"
       [column-multi-sort="{string}"]
       [column-reordering="{string}"]
       column-configuration="{array.<object>}">
</rx-data-table>

Parameters

ParamTypeDetails
pager-object
(optional)
Object

This is the page tracking object for the directive. If no page tracking object is passed in, then the data table will be shown without pagination.

list-of-dataArray.<Object>

This is the list of data that the data table will represent

predicate
(optional)
Array.string=string=boolean

This is the sort predicate. This should be an array of strings that will be used as sort predicates. (i.e. "['-severity']"). You may also pass a value of false in order to disable sorting on all columns that don't have a sortField value explicitely defined.

row-key
(optional)
string

This is the attribute of the data objects that will be used to attatch a data-value-key paramater to each row of the table

row-details
(optional)
function

Setting this attribute to a function will add a toggle to the end of the right side of the table on each row that will insert and fill in the result of this function onto the table.

row-details-clause
(optional)
expression

If this attribute is set, it will be evaluated for every row, and if true, will only apply the row details function to those rows.

item-namestring='Items'

This is what the data table will fill in to indicate what the items in it really are.

notify-duration
(optional)
number

This is a default notification duration in milliseconds. This value is 3000 by default.

data-loading
(optional)
boolean

This is a flag that should be passed to the data table to indicate loading status.

row-style
(optional)
string

This is an object in a string format that is parsed in the code and applied to each row in the table.

It takes three attributes:

  • class {string}: This is the class that gets to the row.
  • field {string}: This is the data field that the comparisons will be made against.
  • bool {boolean}: If true, then it just checks the field to see if it's truthy and applies the class. If it's false (default), then the value of the field is applied as a class along with the value in the class attribute. To account for non-string data values in the field, this will check to see if the first character in the field value is numerical, if it is, it prepends the value with an underscore (_). So, for example, if you have your class as data-service-level and the field value turns out to be a 2, then the class attribute of the row will be class="data-service-level _2". This allows you to grab that row with .data-service-level._2 and apply CSS values to it.

o @param {Object} column-display This object will hold the current display state of the various columns in the data table. It has/needs two properties:

  • index {integer} This is the current column preset index that is selected.
  • config {array.<integer>} This is the list of the currently displayed column indices from the columnConfiguration object
column-presetsarray.<Object>

This is a list of objects that configure the available column presets for the data table. The format of each object should be as follows:

  • title {string} This is the title of the preset that will show in the dropdown
  • config {array.<integer>} This is the list of indices from the columnConfiguration object that will be displayed. Order matters.
column-multi-sort
(optional)
string

This is a string taking value of true or false. If true, the multi-sorting capabilities of the table will be enabled. Value is false by default.

column-reordering
(optional)
string

This is a string taking value of true or false. If true, the columnReordering and presets are enabled. Default is false.

column-configurationarray.<object>

This are the available column definitions, see the extended information for this in the Data Table Guide
